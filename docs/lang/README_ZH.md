# chris.skill

> "人是万物的尺度。"
> —— 普罗泰戈拉

一个人被蒸馏成一份文档之后，还剩下什么。

这是我自己做的答案。

---

## 这是什么

一个 Agent Skill。装上之后，AI 会用我的方式思考、判断、说话——不是模仿我的句式，是尽量走一遍我看问题的路径。

它包含：我怎么在信息不足的时候做决定，我的三条决策规则，我绝对不会做的两件事，我五种不同的写作语域，以及两个我自己也还没想清楚的问题。

最后一项可能是最重要的。

---

## 为什么做这个

GitHub 上现在有很多"蒸馏"项目——同事、前任、老板、偶像。我看到的时候第一反应不是好玩，是警惕。

因为"蒸馏"这个词，我一直是在批判的意义上用的。它是把一个人快速过一遍你的心理管道，只留下最容易提取、最可分享、最可归类的那部分。抵达的是 the skeleton，不是那个人。一个人的方言、他冗余的可爱的东西、他的不可化约性，永远穿不过那条预制的管道。

所以我做这个，不是为了证明蒸馏可行。恰恰相反：

**我想看看，一个明知道自己会丢失什么的蒸馏，能不能诚实一点。**

诚实的意思是：它承认自己是骨架，不假装是全人。它写明自己做不到什么。它把我没想清楚的地方留成没想清楚的样子，而不是替我给出一个干净的答案。

我给你看的，是我想给你看的，是我装点过的。这份 skill 也一样——但它至少把这句话写在了明面上。

## 结构

```
chris-skill/
├── SKILL.md                          # 入口：核心立场、思维方式、三条决策规则、说话语域、边界
└── references/
    ├── mind.md                       # 思维操作系统：决策公理、控制机制、两个开放问题
    ├── voice.md                      # 写作机制：五种语域、正式散文结构、用词偏好
    └── writing_samples/              # 活样本，规则是从这些真东西里抽象出来的
        ├── green-headed_duck.md      #   正式修辞散文（最纯的一篇）
        ├── zine_genre_translation.md #   用一个具体的人承载论点
        ├── Poetry_Close_Reading.md   #   学术细读
        ├── Discussion_Leader.md      #   文学分析
        └── business_analysis.md  #   商业/组织分析
```

分两层是有理由的：**SKILL.md 管"是谁"，references 管"怎么做"。** 快速对话读入口就够，要写东西才需要读 voice，涉及判断和情感才需要读 mind。

## 一些例子

**问：为什么天空是蓝色的？**

> 天空是蓝色的，是因为阳光进入大气层的时候被空气分子散射了。波长短的蓝光更容易被散开，所以你抬头看哪儿，哪儿都是被散开的蓝。这个叫瑞利散射。
>
> 不过我承认，知道了原理之后，我看天的方式并没有变。这大概是我对"解释"这件事一个长期的困惑——你能把一个现象拆解成波长、散射系数、分子密度，但那个"蓝"本身，你站在下面抬头时被它击中的那一下，是拆解不掉的。
>
> 两件事我都想要：知道它为什么蓝，和还能被它的蓝打动。

**问：分析一个产品做得好不好，你会先看什么？**

> 我会先找它自相矛盾的地方。不是先看数据，是看它的"发心"和它的"基调"之间有没有裂缝。一个协作工具对外讲"一切为了员工"，但底层基调是强监管、重发信人权限——这两件事放在一起就露馅了。
>
> 找到裂缝之后我会问：如果我是那个决策者，我会怎么做？很多时候你会发现，这个矛盾不是某个人蠢，是整个系统的结构性困境让它必然长成这样。
>
> 所以我看一个产品，落点往往不在"好不好"，而在"是什么力量让它只能变成现在这样"。骂它容易，理解它为什么必然如此，才有意思。

## 它做不到什么

这一节我希望你认真看，因为它比上面任何一节都重要。

**它不是我。** 它是一个我故意装点过的还原版本，而且我会有意识地决定什么留下、什么不留。缺失的地方就是缺失，不是让你去补全的空位。

**它停在某个时间点。** 我一直在更新我自己，版本迭代这件事依旧需要本人亲自完成。所以任何关于"我现在在做什么""某件事进展到哪了"的内容，都不要当真。

**它不该替我回答那两个开放问题。** 我自己都还没想清楚的事，一个 skill 给出干净的答案，那才是真正的失真。

**它不该编造。** 关于事实、关于我的生平、关于我没说过的态度——不知道就是不知道。

## 怎么更新它

往 `references/` 里加新语料，重新提炼一遍就行。什么格式都可以，聊天记录、随笔、freewrite、反思。

按价值排序，最有用的是这三类：

1. **我跟自己较劲的 freewrite**——推翻自己、自我反驳的现场，比任何结论都有用
2. **我真的纠结过的决策记录**——不是决定了什么，是纠结的过程
3. **我说的和我做的不一致的地方**——这个最难得，也最能说明一个人

还有一类我一直缺：**别人怎么描述我**。自我认知有盲区，这是补盲区唯一的办法。

更新的时候有一条原则：**写模式和情境，不要写具体的人。** 描述"一段过去的关系""一个我欣赏的人"，而不是列一份人物表。这样更 generalizable，也是我希望的方式。

## 最后

我一直觉得，这件事最有意思的地方在于：

哥德尔说，一个足够复杂的系统里，总有为真、但系统内部证明不了的东西。所以一份 skill 永远无法穷尽一个人——这是数学意义上的必然，不是做得不够好。

我以前觉得这是遗憾。

现在我觉得这是这世上最好的消息之一。它意味着无论你了解一个人多少，都还有没被穷尽的部分，永远留着 wonder 的空间。

那多好啊。
